索引重要

Posted on 2013-05-23(周四) in experience

e 的索引本身只有 0～1 

class A {
public:
    element e[2];
    Postman right;
    Postman left;
public:
    void setData();
}

void A::setData() {
    e[1] = e1;
    e[2] = e2;
    right = postman1;
    left = postman2;
}

上面这个看起来没有错，但是当下一次要使用 e的时候会发现它和原来的数据不同

是因为 e本来是2个值的数组，但索引超出了，第二个有值，第一个没有值 e的索引本身只有 0～1

因为 e[2] 侵占了其他数据的内存，当其他数据改变时，e[2] 就跟着改变，但是 e[1] 还是自有内存，不变
